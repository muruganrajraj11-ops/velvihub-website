⚡ RSTP (Rapid Spanning Tree Protocol) – தமிழில் எளிமையாக

RSTP (Rapid Spanning Tree Protocol) என்பது STP-ன் மேம்படுத்தப்பட்ட Version ஆகும்.

Network Loop-களை STP போலவே தடுக்கிறது. ஆனால் மிகவும் வேகமாக Network Recovery செய்ய உதவுகிறது.

🏠 Real-Life Example

ஒரு Road மூடப்பட்டால் பழைய Map மாற்று வழியை கண்டுபிடிக்க நேரம் எடுக்கும்.

Google Maps உடனே புதிய Route காட்டுவது போல RSTP வேகமாக புதிய Path-ஐ தேர்வு செய்கிறது.

⚙️ RSTP எப்படி வேலை செய்கிறது?

RSTP Root Bridge-ஐ தேர்வு செய்து Loop இல்லாத Network Topology உருவாக்குகிறது.

Link Failure ஏற்பட்டால் Alternate Path-ஐ சில விநாடிகளில் Activate செய்கிறது.

📌 RSTP முக்கிய அம்சங்கள்

  • ✅ Fast Convergence
  • ✅ Quick Recovery
  • ✅ Loop Prevention
  • ✅ Better Network Availability
  • ✅ Enterprise Network Support

📊 STP vs RSTP

Feature STP RSTP
Standard IEEE 802.1D IEEE 802.1w
Convergence Time 30–50 Seconds 1–6 Seconds
Recovery Speed Slow Fast
Enterprise Usage Limited Highly Preferred

📌 RSTP Port Roles

  • Root Port
  • Designated Port
  • Alternate Port
  • Backup Port

🎯 Interview Questions

Q: RSTP என்பதன் விரிவாக்கம் என்ன?

A: Rapid Spanning Tree Protocol

Q: RSTP எந்த IEEE Standard?

A: IEEE 802.1w

Q: RSTP ஏன் உருவாக்கப்பட்டது?

A: STP Convergence Time குறைக்க.

🚦 Easy Analogy

STP ஒரு பழைய Traffic System என்றால், RSTP ஒரு Smart Traffic System.

Road Block ஏற்பட்டவுடன் உடனே மாற்று வழி திறக்கும்.

🎯 முடிவு

RSTP என்பது STP-ஐ விட வேகமான மற்றும் திறமையான Loop Prevention Protocol ஆகும்.

இன்றைய Enterprise Networks-ல் RSTP அதிகமாக பயன்படுத்தப்படுகிறது.
